# 小蛋饼导航站 - 开发维护文档

## 一、项目概述

这是一个**纯前端静态导航网站**，所有数据存储在浏览器 localStorage 中，无需后端数据库。

- 🌐 线上地址：https://xiaodanbin.github.io/nav-site/
- 💻 本地地址：http://localhost:3000/
- 📂 源码仓库：https://github.com/xiaodanbin/nav-site

## 三、如何本地开发调试

### 启动本地服务器

```bash
cd "D:\APP-install\66-trae\others\project\web"
npm start
# 或双击运行 启动导航站.bat
```

然后打开浏览器访问 `http://localhost:3000/`

### 每次修改后如何生效

**HTML / JS 文件修改**：直接刷新浏览器即可（需清理 localStorage 才会加载最新数据）。

**db.js 数据文件修改**：需要增加版本号或手动清理 localStorage 才能生效：
1. 打开浏览器 → F12 → Application → Local Storage
2. 删除 `nav_db_version` 条目（或改为更大数字）
3. 刷新页面

## 四、修改指南

| 修改内容 | 修改文件 | 说明 |
|---------|---------|------|
| 修改网站数据 | `js/db.js` | `DEFAULT_CATEGORIES` 存放分类，`DEFAULT_SITES` 存放网站 |
| 修改首页样式 | `index.html` 内联 `<style>` | 搜索框、主布局等 |
| 修改首页逻辑 | `index.html` 内联 `<script>` | 搜索、分类渲染逻辑 |
| 修改管理后台 | `admin.html` | 增删改网站、分类 |
| 修改脚本逻辑 | `js/main.js` | 通用工具函数 |

### 4.1 修改网站数据（db.js）

```javascript
// 分类数据：id 是唯一标识，parent 为 null 是一级分类，有 parent 是二级分类
const DEFAULT_CATEGORIES = [
    { id: 'term-30', name: '常用站点', icon: '⭐', parent: null },           // 一级
    { id: 'term-30-32', name: '阅读', icon: '📖', parent: 'term-30' },     // 二级
];

// 网站数据：category 字段对应上面分类的 id
const DEFAULT_SITES = [
    { id: 1, name: '百度', url: 'https://www.baidu.com', desc: '搜索引擎', icon: '🔍', category: 'term-30-32', tags: ['搜索'], views: 100 },
];
```

**注意**：修改 db.js 后必须：
1. 增加 `DB_VERSION` 版本号（位于文件顶部）
2. 同时更新 `index.html` 中 `db.js` 的 `?v=` 参数

## 五、如何部署到 GitHub Pages

### 5.1 整体流程

```
本地修改 → git commit → git push → GitHub Actions 自动部署 → GitHub Pages
```

### 5.2 具体步骤

```bash
# 1. 进入项目目录
cd "D:\APP-install\66-trae\others\project\web"

# 2. 查看修改了哪些文件
git status

# 3. 添加要提交的文件
git add index.html js/db.js

# 4. 提交（写清楚改动内容）
git commit -m "feat: 新增XX功能"

# 5. 推送到 GitHub（推送到 main 分支）
git push

# 6. 等待约 1 分钟，GitHub Actions 自动构建并部署
```

### 5.3 部署状态查询

- GitHub Actions：https://github.com/xiaodanbin/nav-site/actions
- 绿色 ✅ = 部署成功，红色 ❌ = 失败

### 5.4 常见问题

**Q: 修改了文件但线上没更新？**
A: 可能是浏览器缓存，需要 Ctrl+F5 强制刷新，或清理 localStorage。

**Q: GitHub Actions 部署失败？**
A: 查看 Actions 页面中的错误日志，常见原因：语法错误、文件路径问题。

## 九、技术栈

- **前端**：原生 HTML + CSS + JavaScript（无框架）
- **数据存储**：浏览器 localStorage
- **本地服务器**：Node.js（serve 包）
- **部署**：GitHub Pages + GitHub Actions
- **站点搜索**：实时下拉搜索（搜索全部数据库，非 DOM 可见元素）
